# Phase 6 Plan 03: Add Guided Rerun and Follow-Up Actions That Preserve Provenance and Review State

<objective>
Close the Phase 6 follow-up loop by turning chat-driven next-step decisions into structured, lineage-preserving history.

Purpose: the current repository can already refresh reports after a bounded follow-up execution, but the locked Phase 6 context now requires more than "did a rerun happen?" It needs to record why the user continued, whether the next step is more evidence, parameter correction, study extension, or task completion, and which refreshed report/provenance pair becomes the new evidence anchor. This plan extends the existing follow-up history contract and tool flow without turning it into an autonomous loop controller.
Output: richer follow-up history entries, bounded follow-up tool behavior that preserves decision intent and lineage, and a cockpit summary that shows the latest decision and resulting evidence anchor.
</objective>
