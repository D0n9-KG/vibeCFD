# Phase 6 Plan 02: Surface Supervisor Scientific Gate Decisions and Operator Actions as Lightweight Chat-Driven Workbench State

<objective>
Translate the existing scientific gate and remediation outputs into a lightweight decision context that hands the actual next-step choice back to the main chat flow.

Purpose: the roadmap still needs review and follow-up decisions to be visible in the workbench, but the locked Phase 6 context rejects a separate approval dashboard. This plan creates a structured decision summary, maps scientific gate state into chat-ready next-step options, and renders that context in the existing cockpit without introducing direct action buttons or a workflow shell.
Output: a delivery-decision payload contract, runtime preservation of that contract, and a compact workbench decision surface that explicitly tells the user to continue in chat.
</objective>
