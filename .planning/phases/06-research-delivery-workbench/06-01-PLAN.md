# Phase 6 Plan 01: Build Final Report Packaging With Strong Artifact Linkage and Chinese Research Delivery Format

<objective>
Turn the existing result-report payload into a researcher-facing Chinese delivery artifact that is readable first and traceable second.

Purpose: the repository already emits `final-report.json`, Markdown, HTML, figures, research-evidence summaries, and provenance pointers, but the output still behaves like an accumulated technical payload instead of a conclusion-first research deliverable. This plan restructures the report contract and renderers so the report starts from what can be claimed right now, then proves each claim with inline source references, confidence/claim language, artifact links, and a complete evidence index.
Output: a conclusion-first Chinese final report contract, rendered Markdown/HTML with explicit evidence linkage, and a cockpit view that reads the same structured report sections.
</objective>
