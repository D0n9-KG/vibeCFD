# Phase 8 Plan 01: Unify Skill Creator Orchestration, Thread State, and Package Schema

<objective>
Define the stable lifecycle contract before publish-management and revision features build on top of it.

Purpose: today Skill Studio already generates a complete DeerFlow-compatible package, but the persistent truth is still split between thread-local artifacts and a small `submarine_skill_studio` summary. This plan introduces one durable skill lifecycle record, expands the thread-state contract with asset identity fields, and teaches current frontend helpers to consume that lifecycle shape instead of inferring everything from artifacts alone.
Output: a persisted lifecycle record keyed by skill asset, draft generation that emits lifecycle metadata next to the existing package artifacts, and dashboard/workbench helper code that understands the new contract.
</objective>
