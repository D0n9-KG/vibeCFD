# Phase 8 Plan 02: Add Publish, Auto-Configure, and Management Flows for Project-Local DeerFlow Skills

<objective>
Turn publish from a one-shot install button into a managed project-level application flow.

Purpose: the codebase can already publish a generated `.skill` archive into `skills/custom` and flip the enable bit, but it cannot yet persist management state, list published lifecycle data, or let the user configure explicit step bindings from the current product surface. This plan adds lifecycle-aware publish routes, project-level binding persistence, lead-agent guidance for those bindings, and the frontend controls needed to manage them.
Output: publish and lifecycle management APIs, project-level step-binding guidance for later threads, and workbench controls for enablement plus explicit role bindings.
</objective>
