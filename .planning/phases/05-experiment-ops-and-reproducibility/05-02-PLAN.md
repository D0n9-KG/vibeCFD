# Phase 5 Plan 02: Productize Experiment Manifests, Baseline-versus-Variant Linkage, and Comparison Summaries

<objective>
Extend the experiment layer from deterministic study-only linkage to a unified baseline-plus-variants registry.

Purpose: the current experiment pipeline already handles scientific-study variants well, but it still treats user-authored variants as extra or suspicious. This plan widens the registry contract so baseline, built-in study variants, and custom variants all travel through the same manifest and compare chain.
Output: unified experiment registry contract, linkage rules that understand custom variants, and compact cockpit/report summaries that keep baseline-to-variant lineage readable.
</objective>
