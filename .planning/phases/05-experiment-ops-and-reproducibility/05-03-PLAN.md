# Phase 5 Plan 03: Align Runtime Configuration Across Local, Docker Compose, and Deployment Paths With Recovery Guidance

<objective>
Make environment consistency explicit, measurable, and operator-honest.

Purpose: the current stack can run locally, in Docker Compose dev, and in deployed environments, but it does not yet record which environment profile produced a run or whether that profile matches a supported reproducibility target. This plan introduces normalized fingerprints, parity assessment, downgrade logic, and compact recovery guidance in the existing cockpit/report surfaces.
Output: per-run environment parity assessment, reproducibility downgrade rules, aligned runtime-profile docs/config, and minimal cockpit visibility for drift handling.
</objective>
