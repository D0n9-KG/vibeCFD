# Phase 5 Plan 01: Persist Full Run Provenance and Reproducibility Metadata

<objective>
Create one authoritative provenance entrypoint for every submarine run.

Purpose: the repository already writes `openfoam-request.json`, `solver-results.json`, `study-manifest.json`, `experiment-manifest.json`, and `run-compare-summary.json`, but a researcher still has to infer which file is the canonical rerun record. This plan introduces `provenance-manifest.json`, writes it at solver-dispatch time, preserves the pointer inside `submarine_runtime`, and feeds it into reporting.
Output: canonical provenance manifest, reducer-safe runtime persistence, and report-ready provenance summaries.
</objective>
