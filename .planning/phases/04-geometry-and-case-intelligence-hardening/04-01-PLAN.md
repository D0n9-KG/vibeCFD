# Phase 4 Plan 01: Harden Geometry Inspection With Unit, Scale, and Reference-Value Sanity Checks

<objective>
Promote geometry preflight from a descriptive STL summary into a traceable geometry-trust gate.

Purpose: the current code can parse STL, compute bounds, and guess a length, but it still treats that guess as solver-ready too early. This plan introduces structured geometry findings and makes solver dispatch consume qualified geometry reference values instead of silent heuristics.
Output: geometry-trust artifacts, runtime-visible geometry findings, and solver-dispatch inputs that respect geometry ambiguity.
</objective>
